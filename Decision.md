## Was ist richtig für mich?

* Wenn man in einem größeren Unternehmen arbeitet, das schon Geschäftsbeziehungen mit Microsoft hat, 
  könnte GitHub Copilot eine gute Wahl sein.
  * Solid Basis
  * Kostengünstig
  * Bedenken: Erweiterungen für viele Editoren
* Wenn wir nicht .NET entwickeln und etwas haben wollen, das uns mehr Möglichkeiten bzgl. LLM-
  Konfiguration gibt, dann könnte Cursor eine gute Wahl sein.
  * Gesamtes Kontextfenster ist leicht ausnutzbar
  * "Fast"-Modus, Kontrolle über "Thinking"
  * Bedenken: Nur ein Editor (nämlich Cursor)
* Claude Code
  * Innovativ
  * TUI oder GUI - beides ist super
  * ABER: Nur Anthropic-Modelle
  * Kostengünstig, wenn nicht über API abgerechnet wird
  * Gebündelt mit Chat Client
* OpenAI Codex
  * Innovativ, aber anders als Anthropic
  * TUI oder GUI - beides ist super
  * ABER: Nur OpenAI-Modelle
  * Kostengünstig, wenn nicht über API abgerechnet wird (günstiger als Anthropic)
  * Gebündelt mit Chat Client
* Was ist mit Google? Top, aber aus Zeitgründen nicht in diesem Classroom behandelt.
* OSS-Optionen
  * Kilocode: Top GUI
  * OpenCode: Top TUI
  * Beide: Offenheit bzgl. LLMs
  * Volle Unterstützung lokaler LLMs
